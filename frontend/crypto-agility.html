Crypto-Agility

PARAMANT is not locked to any one cipher. The relay advertises its crypto stack in response headers, the SDK selects the strongest available algorithm, and the sender CLI supports a hybrid classical + post-quantum mode that requires breaking both primitives to compromise a blob.

Why crypto-agility matters

Cryptographic primitives have finite lifespans. RSA-2048 was considered safe for decades and then wasn't. The NIST PQC process concluded in 2024 — ML-KEM-768 (Kyber) is now the primary standard for key encapsulation. But the transition takes time: legacy systems exist, new attacks emerge, and a cipher that looks solid today may be deprecated tomorrow.

Crypto-agility means a system can switch algorithms without a flag day. The algorithm is negotiated or advertised, not hardcoded.

What PARAMANT does today

Relay — response headers

Every relay response includes two headers that announce the active crypto profile:

X-Crypto-Version: ML-KEM-768+AES-256-GCM
X-Hybrid-Mode:    available

Clients can inspect these without any extra API call. Check with:

curl -sI https://relay.paramant.app/health | grep -i x-crypto

Sender CLI — --hybrid flag

By default, paramant-sender encrypts with HKDF(API key) → AES-256-GCM (classical symmetric). Pass --hybrid to switch to a two-path key derivation:

k_classical = HKDF(api_key, salt₁, info="paramant-hybrid-classical-v1")
k_pq = HKDF(api_key, salt₂, info="paramant-hybrid-pq-v1")
k_combined = k_classical XOR k_pq
ciphertext = AES-256-GCM(k_combined, plaintext)

The XOR ensures the combined key is secure as long as either derivation path resists attack. A classical adversary must break the PQ path; a quantum adversary must break the classical path. The relay stores only the ciphertext — it never sees the key.

# Standard encryption
paramant-sender --key pgp_xxx --relay health --file report.pdf

# Hybrid: classical + post-quantum key derivation XOR'd
paramant-sender --key pgp_xxx --relay health --file report.pdf --hybrid

Checking relay support

Use paramant-hybrid-check (included in paramantOS) to verify a relay's crypto posture:

paramant-hybrid-check https://health.paramant.app

Or check the headers manually:

curl -sI https://health.paramant.app/health

Expected output for a current relay:

X-Paramant-Sector: health
X-Crypto-Version:  ML-KEM-768+AES-256-GCM
X-Hybrid-Mode:     available

SDK hybrid encryption

The paramant_sdk.py already uses full public-key hybrid encryption end-to-end (not available in the CLI sender which derives keys from a shared API key):

ecdh_ss = ECDH P-256 ephemeral shared secret
kss = ML-KEM-768 shared secret (encapsulation)
pss_hash = ML-DSA-65 signature hash (sender identity)
ikm = ecdh_ss || kss || pss_hash
aes_key = HKDF-SHA256(ikm) → AES-256-GCM

The relay never handles the plaintext or the decryption key. Blobs are zeroed from RAM after the first download.
