No VPN. No network changes. Deploys on a Raspberry Pi. Data is destroyed after delivery.
The problem
  • OT zones cannot connect directly to IT/cloud
  • VPNs create persistent, auditable attack surfaces
  • File shares leave data at rest after delivery
  • Legacy protocols (FTP, SMB) are not quantum-safe
  • Traffic size reveals firmware push intent to DPI
Ghost Pipe solves this
  • Quantum-safe data diode for the OT/IT boundary
  • No persistent connection between zones
  • Burn-on-read: data destroyed after first download
  • ML-KEM-768 + ML-DSA-65 (NIST FIPS 203/204)
  • Fixed 5 MB padding: every transfer looks identical
