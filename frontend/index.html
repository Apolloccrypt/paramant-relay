Protocol

Zero plaintext. Zero storage. Zero trust.

Every blob is end-to-end encrypted before leaving your device. The relay sees only fixed-size, indistinguishable ciphertext.

Post-quantum hybrid

ML-KEM-768 and ECDH P-256 in a single handshake. Resistant to both classical and quantum adversaries. NIST FIPS 203 certified.

ML-KEM-768 + ECDH

Burn-on-read

Data is overwritten with random bytes immediately after the first retrieval. No disk writes, no persistence, no recovery after delivery.

RAM-only

Sender authentication

Every blob signed with ECDSA P-256 before upload. ML-DSA-65 post-quantum signatures supported. Retry with exponential backoff.

ECDSA + ML-DSA-65

Global anycast

Five Fly.io nodes across Amsterdam, Frankfurt, Stockholm, Paris, and Chicago. Minimum three always active. Session affinity per API key.

5 nodes · EU primary

Real-time push

WebSocket stream at /v2/stream delivers blob_ready events without polling. NATS JetStream backend. Sub-100ms for IoT and medical vitals.

WebSocket · NATS

Merkle audit chain

Every inbound and outbound event chained in a tamper-evident log. Verifiable chain integrity. Export as JSON or CSV for compliance.

Merkle · CT log
Identity & PKI

No central database. Every device proves itself.

Devices authenticate with cryptographic proof instead of shared API keys. The relay does not need to know who you are.

W3C Decentralized Identity

Each device gets a DID instead of a shared key. Authentication works by signing the request. No central users.json, no single point of failure.

did:paramant:xxx

Certificate Transparency log

Every public key registration logged in a public Merkle tree. Anyone can verify a device is registered without seeing any data. Inclusion proofs on demand.

Public · /v2/ct/log

Hardware attestation

TPM 2.0 and Apple Secure Enclave sign the sender public key. The relay verifies the key came from real hardware, not a software fake.

TPM 2.0 · Secure Enclave
Use cases

Built for regulated industries.

Where a data breach has legal or medical consequences — and where proving data was not retained matters as much as the transfer itself.

Healthcare

DICOM files from MRI and CT scanners to PACS systems. HL7 FHIR resources between EPD systems. Patient vitals from ambulance to hospital. No data-at-rest, NEN 7510 ready.

Legal & notary
