Wat is Ghost Pipe?

Ghost Pipe is een eenmalige versleutelde bestandsoverdracht. De relay slaat nooit iets op schijf op — alleen in RAM, maximaal 300 seconden. Zodra de ontvanger het bestand ophaalt, wordt de kopie definitief vernietigd (blob.fill(0) + delete).

EigenschapWaarde
EncryptieML-KEM-768 (NIST FIPS 203) + ECDH P-256 hybrid
BlokgrootteAltijd exact 5.242.880 bytes (metadata masking)
TTL300 seconden in RAM — daarna automatisch vernietigd
Burn-on-readTweede ophaalpoging → HTTP 404
Relay locatieHetzner DE (Neurenberg) — GDPR, geen US CLOUD Act
OpslagRAM-only — nooit op disk

Ghost Pipe Protocol

Wire format

Elk versleuteld pakket heeft het volgende formaat vóór padding:

[4B]    Magic: "PQHB"
[65B]   Ephemeral ECDH P-256 public key
[1088B] ML-KEM-768 ciphertext
[12B]   AES-GCM nonce
[NB]    AES-256-GCM ciphertext + 16B authentication tag

5MB padding

Na encryptie wordt het pakket gepadded naar exact 5.242.880 bytes:

[8B]    Originele data lengte (big-endian uint64)
[NB]    Wire format blob
[RB]    Cryptografisch random padding
= exact 5.242.880 bytes (altijd)
Waarom vaste blokgrootte? DPI-systemen kunnen verkeerspatronen analyseren. Als elke transfer precies 5MB is, ziet niemand of het een SMS of een rechtbankdossier is.

Key derivation

ikm  = ECDH_shared_secret || ML-KEM_shared_secret
key  = HKDF-SHA256(ikm, salt=eph_pub, info="paramant-ghost-pipe-hybrid-v1")
     → 32 bytes AES-256 sleutel

Bestand versturen

POST/v2/inbound

Twee upload methoden — kies op basis van omgeving:

JSON + base64 (standaard, browser/SDK)

{
  "hash":    "sha256_van_ontvanger_pubkey_plus_nonce",
  "payload": "base64_encoded_5mb_blob",
  "api_key": "pgp_..."
}
Content-Type: application/json

Binary upload (IoT/CLI — 33% minder bandbreedte)

curl -X POST https://relay.paramant.app/v2/inbound \
  -H "Content-Type: application/octet-stream" \
  -H "X-Hash: a3f8c2d1..." \
  -H "X-Api-Key: pgp_..." \
  --data-binary @bestand.gpb

# Geen base64 overhead — raw 5MB binary
# X-Hash: sha256 hex van de blob routing hash
Hash privacy: De hash is sha256(pubkey_bytes || 16_random_bytes) — niet sha256(pubkey). Een nonce maakt de hash onvoorspelbaar. Zonder nonce kan een observeerder de /v2/status endpoint pollen en communicatiefrequentie achterhalen. De nonce is ingebakken in de CLI en SDK — niets om zelf te doen.

Response

{
  "ok":          true,
  "hash":        "a3f8c2d1e7b4...",
  "size":        5242880,
  "ttl_ms":      300000,
  "expires_at":  1234567890123,
  "burn_on_read": true,
  "plan":        "pro"
}
Hash privacy: Gebruik niet puur sha256(pubkey) als hash — dit laat derden de /v2/status endpoint pollen om communicatiefrequentie te observeren. Voeg een nonce toe: hash = sha256(pubkey + ':' + random_nonce) en stuur de nonce mee met de hash naar de ontvanger.

CLI

./paramant send document.pdf \
  --to 04abc...1249hexchars \
  --api-key pgp_... \
  [--json]

Python SDK

from paramant_ghostpipe import send

result = send(
    data=open("document.pdf", "rb").read(),
    recip_pub_hex="04abc...1249hexchars",
    api_key="pgp_...",
    relay_url="https://relay.paramant.app"
)
print(f"Hash: {result['hash']}")

Bestand ontvangen (burn-on-read)

GET/v2/outbound/:hash

Geeft de raw binary blob terug (5.242.880 bytes). Relay vernietigt de blob direct na verzending.

curl -s "https://relay.paramant.app/v2/outbound/a3f8c2d1..." \
  --output bestand.enc

# Tweede request:
curl -s "https://relay.paramant.app/v2/outbound/a3f8c2d1..."
# → HTTP 404 {"error":"Niet gevonden of al opgehaald"}

ACK mechanisme

Zodra de relay de stream start, wordt de entry uit gpMempool verwijderd — een tweede GET geeft direct 404. De originele blob blijft intern 30 seconden in pendingBurn voor fysieke cleanup:

  • Client stuurt POST /v2/ack/:hash → blob.fill(0) + delete direct
  • Geen ACK binnen 30s → timeout → blob.fill(0) + delete automatisch

Dit is geen "soft" burn — data is direct onbereikbaar. De 30s window is alleen voor gegarandeerde zero-overwrite na een geslaagde stream.

POST/v2/ack/:hash
{
  "ok": true,
  "destroyed": true
}

Status checken

GET/v2/status/:hash
{"available": true, "hash": "a3f8..."}

Grote bestanden (>5MB)

Pro: max 500MB (100 blokken). Kantoor: max 2GB (400 blokken).

POST/v2/manifest
# Stap 1: registreer manifest
POST /v2/manifest
{
  "total_blocks":   10,
  "recipient_hash": "sha256...",
  "api_key":        "pgp_..."
}
# Response: { "manifest_hash": "abc123...", "total_blocks": 10 }

# Stap 2: stuur elk blok
# Relay berekent block-hash automatisch via X-Block-Index header
# Developer hoeft geen hash-logica te implementeren

POST /v2/inbound
Content-Type: application/octet-stream
X-Api-Key: pgp_...
X-Manifest-Hash: abc123...
X-Block-Index: 0
Body: <raw 5MB blob>

# OF via JSON:
{
  "manifest_hash": "abc123...",
  "block_index":   0,
  "payload":       "base64_5mb_blok",
  "api_key":       "pgp_..."
}
# hash wordt automatisch berekend: sha256("abc123...:0")
Limiet: Gratis transfers zijn niet mogelijk — Ghost Pipe vereist altijd een API key.

Sidecar proxy (legacy devices)

Legacy apparatuur (MRI scanners, PLC controllers, SCADA systemen) kan Ghost Pipe gebruiken zonder firmware aanpassing. De sidecar proxy ontvangt gewone HTTP en maakt Ghost Pipe transfers.

# Start proxy
./paramant proxy --listen 0.0.0.0:8765 --to <ontvanger_pubkey> --api-key pgp_...

# Legacy device stuurt:
POST http://localhost:8765/send
Content-Type: application/octet-stream
Body: <raw data>

# Antwoord:
{"ok": true, "hash": "a3f8...", "ttl_ms": 300000}

# Health check:
GET http://localhost:8765/health
{"ok": true, "mode": "sidecar", "relay": "https://relay.paramant.app"}

Docker sidecar

services:

  paramant-sidecar:
    image: ghcr.io/apolloccrypt/paramant-agent:latest

Heartbeat / Dead-man-switch

Stuur een versleuteld heartbeat elke N seconden. Het pakket is niet te onderscheiden van een echte transfer — ook een heartbeat is een 5MB blob.

./paramant heartbeat \
  --to <monitor_pubkey> \
  --interval 60 \
  --device-id mri-kamer-3 \
  --alarm-url https://ops.ziekenhuis.nl/alert \
  --api-key pgp_...

# Na 3 gemiste pings:
# [ALARM] 3 heartbeats gemist voor mri-kamer-3
# POST https://ops.ziekenhuis.nl/alert {"device_id":"mri-kamer-3","missed":3}

Python SDK

ML-KEM-768 in Python: De standaard cryptography library ondersteunt ML-KEM-768 nog niet. Gebruik pqcrypto (wrapper om liboqs) voor post-quantum operaties. 
# Voor volledig hybride ML-KEM-768 + ECDH P-256:
pip install pqcrypto cryptography requests

# Alternatief via liboqs Python bindings:
pip install liboqs-python  # vereist liboqs C library

# Controleer beschikbaarheid:
python3 -c "import pqcrypto.kem.kyber768; print('ML-KEM-768: OK')"
from paramant_ghostpipe import HybridKeypair, send, receive, check_status, verify_key

# Keypair genereren
keys = HybridKeypair()
keys.save("paramant_keys.json")
print(keys.public_key_hex)  # 2498 hex chars

# Versturen
result = send(
    data=open("document.pdf", "rb").read(),
    recipient_pub_hex=keys.public_key_hex,
    api_key="pgp_...",
    relay_url="https://relay.paramant.app"
)
print(f"Hash: {result['hash']}, TTL: {result['ttl_ms']}ms")

# Ontvangen (burn-on-read)
data = receive(result["hash"], keys.private_key_hex)
open("ontvangen.pdf", "wb").write(data)

# Status
status = check_status(result["hash"])
print(status["available"])  # True/False

# Key valideren
info = verify_key("pgp_...")
print(info["plan"], info["used"], "/", info["limit"])

Enterprise — Self-hosted relay

Voor organisaties die data nooit via externe servers willen sturen. Vereist Kantoor of Enterprise plan.

# docker-compose.yml
services:
  paramant-relay:
    image: ghcr.io/apolloccrypt/paramant-relay:latest
    restart: unless-stopped
    ports:
      - "127.0.0.1:3001:3001"
    environment:
      PARAMANT_LICENSE_KEY: "pgp_..."   # Kantoor/Enterprise key
      ADMIN_TOKEN:          "eigen_token"
      USERS_FILE:           /data/users.json
    volumes:
      - paramant_data:/data

volumes:
  paramant_data:
Heartbeat licentie (standaard): De relay pings relay.paramant.app elke 24u. Na 3 mislukte checks: read-only modus. Niet geschikt voor air-gapped omgevingen.

Air-gapped licentie (Enterprise)

Voor omgevingen zonder uitgaande internetverbinding (defensie, rechtbank, kritieke infra). Geen heartbeat nodig — relay verifieert een getekend licentiebestand lokaal.

# Stap 1: vraag licentiebestand aan bij Paramant
# Wij genereren een getekend bestand geldig voor 1 jaar:
curl -X POST https://relay.paramant.app/admin/generate-airgap-license \
  -H "Content-Type: application/json" \
  -d '{"admin_token":"...","plan":"enterprise","valid_days":365,"label":"Rechtbank NL"}'

# Response:
{
  "license": "eyJwbGFuIjoiZW50ZXJwcmlzZSI...<base64>.<hmac>",
  "expires": "2027-03-29T00:00:00.000Z"
}

# Stap 2: stel in op self-hosted relay
echo "PARAMANT_AIRGAP_LICENSE=eyJwbGFuIjoiZW50ZXJwcmlzZSI..." >> .env
# Geen PARAMANT_LICENSE_KEY nodig — geen uitgaande verbinding
# Eigen relay URL gebruiken in CLI
./paramant send document.pdf \
  --to <pubkey> \
  --api-key pgp_... \
  --relay https://relay.uwbedrijf.nl
