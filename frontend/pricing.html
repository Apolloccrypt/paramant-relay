Limits

Every tier stores transfers in RAM with a 5 MB default per file. What differs by tier is how long the link lives, how many times it can be read, and how many devices can register for verified transfers. Enterprise customers running a dedicated relay can configure the 5 MB default upward within the RAM budget of their deployment.

Encryption

Every tier uses the same cryptography. Anonymous transfers use AES-256-GCM with a browser-generated key that lives only in the URL fragment. Verified transfers via ParaShare use ML-KEM-768 plus ECDH P-256 hybrid key exchange with ML-DSA-65 signed receipts. There is no lesser encryption at lower tiers. Cryptography is not a paywalled feature at Paramant.

Compliance

All tiers meet NIS2 and GDPR requirements by design. NEN 7510, eIDAS, and IEC 62443 mappings ship with Pro and Enterprise. A signed Data Processing Agreement under GDPR Article 28 is available on request for all paid tiers.

Support

Free tier is self-service via public documentation and the CT log. Pro tier includes email support with published response targets. Enterprise customers get a named contact, a 99.9% SLA, priority incident handling, and deployment assistance for air-gapped and regulated environments.

Frequently asked questions

What happens to my file after the recipient downloads it?

All files are RAM-only. They exist only in server memory and are wiped immediately on download (burn-on-read) or on TTL expiry. Nothing persists after expiry on any tier. There is no disk storage.

Why is the file size limited to 5 MB?

RAM-only storage is finite. 5 MB covers every document that matters: contracts, medical referrals, legal filings, sensor payloads, firmware chunks. The limit is a consequence of the architecture, not an upsell gate. Enterprise customers with dedicated relays can configure higher limits.

Does the anonymous /send flow use ML-KEM-768?

No. The anonymous send flow (/send, no account) uses AES-256-GCM with a browser-generated key delivered in the URL fragment. The key never reaches the relay. ML-KEM-768 + ECDH P-256 hybrid key exchange is used by ParaShare — the registered-device, verified-transfer flow.

Can I self-host Paramant for free?

Yes. The relay is available on GitHub under BUSL-1.1. Community self-hosting is free forever. The license prevents competitors from reselling the relay as a managed service without a commercial license. See the license page for full terms.

Is there a free trial for Pro?

Pro is not yet launched. You can join the waitlist at privacy@paramant.app. In the meantime, create a free account — trial plan includes relay access, no credit card required.

Do all plans include a GDPR Data Processing Agreement?

The standard DPA is publicly available at /dpa and applies to all plans. Enterprise customers can negotiate custom DPA terms as part of their contract.

What does "IEC 62443 / NEN 7510 / NIS2 compliance documentation" mean?

Architecture documents, security controls mappings, and technical reference materials aligned to these frameworks. This is documentation you can use as input for your own compliance process. Paramant does not hold third-party certification for these frameworks.

What happens if Paramant shuts down?

Paramant is published under BUSL-1.1. If the managed service ever closes, every self-hosted relay keeps running indefinitely. The source code is permanently available on GitHub. No vendor lock-in by design.
