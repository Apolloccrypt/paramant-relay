Command-line tools that ship their result through the Paramant post-quantum relay (ML-KEM-768 + ML-DSA-65, ciphertext-only, burn-on-read) using only the public paramant-sdk API.

This page is provisional. The tools below are starter / POC and are not yet published to a package index. The install and usage lines are placeholders that mirror the intended shape (one-line install, three-line usage) and will be replaced as each tool ships.

Authenticate once

# 1. https://paramant.app/signup  →  2. Dashboard → Developer → copy pgp_ key
export PARAMANT_API_KEY=pgp_...
export PARAMANT_DEVICE=my-laptop          # optional, stable device id

paramant-dev-transfer

WeTransfer for the terminal. Send a file, get a token; the other side receives it by token. The relay only ever holds ciphertext, in RAM, and burns it on first read.

pipx install paramant-dev-transfer
paramant-dev-transfer ./build.zip
# → token: pdt_… (share it)

paramant-s3-migrate

Move an S3 object to a device, encrypted, without leaving a copy behind. Bytes are encrypted on your machine; the relay holds only ciphertext.

pipx install paramant-s3-migrate
paramant-s3-migrate s3://bucket/key --to ops@host

paramant-db-backup

One-command off-site database backups that are encrypted before they leave the box. Hands a pg_dump/mysqldump straight to the relay.

pipx install paramant-db-backup
paramant-db-backup --pg "postgres://…" --to backup@vault

paramant-git-archive

Send a snapshot of a repo at any ref, encrypted, without a remote. git archive the tree; the other side picks it up by hash.

pipx install paramant-git-archive
paramant-git-archive HEAD --to teammate@laptop

paramant-docker-migrate

Move a Docker volume to another host, encrypted, without a shared registry or a cleartext SSH pipe. Export, ship, untar on the far side.

pipx install paramant-docker-migrate
paramant-docker-migrate my-volume --to operator@host2

paramant-secrets-sync

Team secrets sync without a cloud vault. Push .env.prod to a teammate; they pull it back by hash. Burned on first read.

pipx install paramant-secrets-sync
paramant-secrets-sync push .env.prod --to alice@laptop

paramant-ci-artifact

Ship CI/CD build artifacts to a destination, encrypted, straight from the pipeline. No public artifact store, no plaintext copy in a bucket.

pipx install paramant-ci-artifact
paramant-ci-artifact ./dist --to deploy@prod

paramant-log-ship

Tamper-evident log shipping for the SOC. Tail a log and ship each batch to your SIEM as an ML-DSA-65-signed blob the auditor can verify.

pipx install paramant-log-ship
paramant-log-ship /var/log/app.log --to siem@collector

paramant-package-sign

Code-sign a release artifact with post-quantum ML-DSA-65, and catch a swap. Produces a detached .psign sidecar (SHA-256 + signature + public key).

pipx install paramant-package-sign
paramant-package-sign dist/app-1.0.tar.gz
# → dist/app-1.0.tar.gz.psign

paramant-db-replicate

Cross-region database replication, post-quantum encrypted. Continuously feed new rows of a Postgres table to another region as encrypted batches.

pipx install paramant-db-replicate
paramant-db-replicate --table orders --to eu-west@replica

Tools are documented in paramant-solutions/tools/. Each ships only ciphertext through the relay and uses the public paramant-sdk API. Questions: read the docs.
