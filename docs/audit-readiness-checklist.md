# Audit-readiness checklist (pre-Cure53 / NCC Group)

First-pass checklist for an external crypto/security audit of paramant-relay +
paramant-core. Generated by the overseer session from the cross-repo consistency
check; treat as a living document. Status reflects the 2026-05-27 snapshot.

## Scope definition
- [ ] Define audit scope: relay transport + routing + storage, core crypto crate,
      or both. Recommend both, with core as the deep crypto target.
- [ ] Freeze a commit/tag per repo for the audit and record bundled
      @paramant/core version (see cross-repo-coordination.md).

## Doc coherence
- [ ] Reconcile version story before audit: relay.js:31 VERSION='3.0.0' is
      emitted by /health while marketing is 2.5.0 and 3.0.0 is reserved for
      ParaSign GA (PROJECT-STATUS.md finding #1). Auditors will flag a service
      advertising a version that does not match its docs.
- [ ] Resolve core docs/architecture.md (lowercase, v0.1 stub) vs ARCHITECTURE.md
      duplicate -- one authoritative file (finding #3).
- [ ] Bump install.sh VERSION pin off the stale v2.4.5 (finding #2).
- [ ] Confirm README /health example matches actual emitted output.

## ADR / decision trail
- [ ] paramant-core ADRs 0001-0021 contiguous -- present.
- [ ] paramant-relay ADRs R001-R0NN contiguous -- R001-R007 present, R008 in
      PR #41, R009/R010 in the overseer PR. Confirm no gaps at audit-freeze.
- [ ] Each crypto-relevant decision (algorithm choice, KDF params, constant-time
      policy) maps to a core ADR -- spot-check 0008/0011/0012.

## Threat model
- [ ] core/docs/threat-model.md reviewed and dated within the audit window.
- [ ] Blind-store non-PQHB rationale (R004) reflected in the threat model.
